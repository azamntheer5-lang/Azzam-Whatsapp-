# ماسح إيصالات واتساب (مشروع شخصي - Android)

## ⚠️ اقرأ هذا أولاً
هذا الكود كامل ومكتوب بعناية، لكنه لم يُختبر بالتصريف الفعلي (compile) لأن بيئة إنشائه لا تملك Android SDK. الأرجح أنه يعمل كما هو أو بأقل تعديل، لكن من المتوقع احتمال ظهور طلب تحديث نسخة مكتبة واحدة عند أول Gradle Sync.

## خطوات التشغيل
1. افتح مجلد المشروع في Android Studio (أحدث نسخة مستقرة) → File > Open
2. انتظر Gradle Sync؛ إن اقترح تحديث نسخة أي مكتبة (AGP/Kotlin/غيرها)، اقبل الاقتراح
3. وصّل هاتفاً حقيقياً (وليس محاكياً - مسارات واتساب ومراقبة الملفات لا تعمل بشكل واقعي على المحاكي) وفعّل "خيارات المطورين" ثم "تصحيح USB"
4. اضغط Run ▶️
5. عند فتح التطبيق أول مرة: اضغط "منح الصلاحية" → في شاشة إعدادات أندرويد التي تفتح، فعّل "السماح بإدارة كل الملفات" لهذا التطبيق
6. ارجع للتطبيق - ستبدأ المراقبة تلقائياً، ويمكنك أيضاً الضغط على زر "مسح الآن" العائم لمسح فوري يدوي

## الميزات الجديدة

**📊 تحليلات ورسوم بيانية** (قائمة ⋮ أعلى الشاشة → تحليلات): رسم بياني عمودي بالإجمالي الشهري، ورسم دائري لأكثر 5 جهات (مرسل/مستلم) تكراراً حسب المبلغ.

**📄 تصدير Excel/PDF**: من نفس القائمة، يصدّر CSV (يفتح مباشرة في Excel/Sheets) أو تقرير PDF منسّق بكل السجلات + الإجمالي، ويفتح شاشة "مشاركة" أندرويد لحفظه أو إرساله.

**🔍 بحث + تعديل يدوي**: مربع البحث أعلى القائمة يفلتر بالاسم/البنك فوراً (والإجمالي المعروض يتحدّث حسب نتائج البحث). اضغط أي سجل لتعديل حقوله يدوياً (مفيد عندما يخطئ الاستخراج التلقائي)، أو اضغط مطولاً للحذف السريع.

**☁️ نسخ احتياطي محمول**: بعكس تشفير التطبيق الأساسي (مرتبط بجهازك فقط عبر Android Keystore)، هذه النسخة محمية بكلمة مرور تختارها أنت، فتقدر تستعيدها على جهاز آخر أو بعد تهيئة الهاتف. **احفظ كلمة المرور جيداً - لا يوجد "نسيت كلمة المرور".**

**🏠 ودجت الشاشة الرئيسية**: اضغط مطولاً على الشاشة الرئيسية → ودجت → ابحث عن "ماسح إيصالات واتساب" لإضافته. يعرض الإجمالي الحالي وعدد الإيصالات، ويفتح التطبيق عند الضغط عليه، ويتحدّث تلقائياً مع أي سجل جديد.

## نقاط مهمة يجب معرفتها

**لن يعمل عبر Google Play** - يحتاج تثبيتاً مباشراً (كما تفعل من Android Studio) لأن صلاحية "كل الملفات" مقيّدة هناك. هذا التطبيق مصمم أصلاً للاستخدام الشخصي غير المنشور فقط.

**العربية**: محرك القراءة الأساسي (ML Kit) لا يقرأ نصاً عربياً من الصور. لهذا أضفنا طبقة ثانية اختيارية: عند فشل/نقص استخراج الاسم عربياً، يستدعي التطبيق تلقائياً Claude API (يقرأ الصورة/PDF مباشرة، بدقة عالية بالعربية) **فقط إن أدخلت مفتاح API** من زر "مفتاح API" أعلى الشاشة. بدون مفتاح، يعمل التطبيق بالـ Regex المحلي فقط (سيستخرج المبلغ/التاريخ عادة، وقد يفوّت الاسم العربي).

- احصل على مفتاح من console.anthropic.com (يتطلب حساباً وبطاقة دفع، والاستخدام مدفوع لكل إيصال يحتاج المساعدة السحابية فقط - ليس على كل ملف)
- المفتاح يُخزَّن مشفَّراً على جهازك فقط (نفس آلية AES-256/Keystore)، ويُستخدم من التطبيق مباشرة بلا خادم وسيط - مقبول لمشروع شخصي، لكن **لا تشارك هذا التطبيق أو الـ APK مع أي أحد بعد إدخال مفتاحك**
- لماذا ليس Gemini Nano (المجاني والمحلي)؟ لأن Google تمنع استخدامه من أي سياق خلفي (خدمة أو Worker) - يعمل فقط والتطبيق مفتوح بالواجهة الأمامية، وهذا يعطّل الغرض الأساسي من هذا التطبيق (الاكتشاف التلقائي في الخلفية). يبقى خياراً جيداً لو حوّلت التطبيق لاحقاً لنمط "افتح وامسح يدوياً" فقط

**البنوك المدعومة بالـ Regex حالياً**: الراجحي وSTC Pay + محلل عام احتياطي (`GenericParser`) يحاول استخراج أي مبلغ/تاريخ حتى لو لم يتعرف على البنك. لإضافة بنكك: أنشئ صنفاً جديداً في `parser/` يطبّق `BankReceiptParser`، وأضفه لقائمة `ParserRegistry.kt`.

**استثنِ التطبيق من توفير البطارية** يدوياً من إعدادات الهاتف (الإعدادات > التطبيقات > هذا التطبيق > البطارية > "بلا قيود") - خصوصاً على أجهزة Xiaomi/Oppo/Samsung ذات القيود المشددة على التطبيقات الخلفية، وإلا فقد يوقف النظام خدمة المراقبة رغم كل الكود.

**إن لم تُكتشف إيصالاتك تلقائياً**: تحقق من المسار الفعلي لمجلد وسائط واتساب على جهازك عبر أي مدير ملفات، وعدّل قائمة `WHATSAPP_PATHS` في `ReceiptWatcherService.kt` إذا اختلف عن المسارات الافتراضية المكتوبة.

**التخزين**: كل السجلات مشفّرة بـ AES-256-GCM عبر مفتاح محفوظ في Android Keystore (محمي بالهاردوير غالباً)، في ملف واحد بمسار `filesDir/transfers.enc`. لا نسخ احتياطي سحابي لهذا الملف (`allowBackup=false` + `data_extraction_rules.xml`).

**حذف سجل**: اضغط مطولاً (long press) على أي بطاقة في القائمة لحذفها.

## هيكل المشروع
```
app/src/main/java/com/example/receiptscanner/
├── MainActivity.kt              # الشاشة الرئيسية + تدفق طلب الصلاحية
├── ReceiptWatcherService.kt     # المراقبة الفورية (Foreground Service + FileObserver)
├── PeriodicScanWorker.kt        # المسح الاحتياطي الدوري كل 15 دقيقة
├── model/Transfer.kt            # نموذج بيانات السجل الواحد
├── ocr/                         # PDF->صور (ML Kit) + CloudExtractor (Claude API الاحتياطي)
├── parser/                      # محلّلات البنوك القابلة للتوسّع + المسجّل
├── processing/                  # الفلترة (نوع/حجم) + خط المعالجة الكامل (محلي ثم سحابي عند الحاجة)
├── export/                      # تصدير CSV وPDF
├── backup/                      # نسخ احتياطي/استعادة محمي بكلمة مرور
├── widget/                      # ودجت الشاشة الرئيسية
├── storage/                     # التشفير + مخزن البيانات + مفتاح API + تتبع الملفات المعالَجة
└── ui/                          # ViewModel + Adapter + شاشة التحليلات
```

## أين تذهب من هنا؟
- **بنوك جديدة**: صنف جديد في `parser/` يطبّق `BankReceiptParser`
- **دقة أعلى للعربية**: جرّب موديل مختلف في `CloudExtractor.kt` (السطر `MODEL`)، أو حسّن نص الطلب (prompt) في نفس الملف
- **تقليل تكلفة السحابة**: عدّل شرط `needsCloudHelp` في `ReceiptProcessor.kt` ليكون أكثر تحفّظاً (مثلاً: تجاهل الاسم تماماً والاكتفاء بالمبلغ/التاريخ فقط)
- **نشر تجاري لاحقاً؟** راجع الدليل التقني المُرسَل سابقاً في المحادثة - يغطي بالتفصيل نقل مفتاح الـ API لخادم خلفي، وقيود Google Play على MANAGE_EXTERNAL_STORAGE
